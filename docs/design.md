# 設計メモ

Windows Core Audio API の `IAudioSessionManager2` を使い、既定の再生デバイスにあるアクティブなオーディオセッションを取得する。各セッションの `ISimpleAudioVolume` で音量とミュートを変更するため、外部ライブラリは不要。

一覧は 2 秒ごとに取り直し、再生開始・終了したアプリを反映する。ウィンドウの最前面状態は UI から切り替えられる。

有効な再生エンドポイントは `IMMDeviceEnumerator.EnumAudioEndpoints` で列挙し、選択したエンドポイントからセッションとマスター音量を取得する。デバイス・セッションとも `IAudioMeterInformation` のピーク値を 0.1 秒間隔で反映し、UI のレベルバーに表示する。

画面は DAW 風のダークグレー基調とし、操作・ピークメーターのアクセントカラーにはオレンジを用いる。上部のアプリメニューを左クリックすると、更新、常に手前、最小化、閉じるを実行できる。

ミキサーは縦に積み重ねたコンパクトなチャンネルカードで表示する。各カードには `73%: MASTER` のようにタイトル先頭へ音量を表示し、横方向の音量フェーダー、音量（グレー）とピーク（オレンジ）を重ねて表示する横方向のレベルメーター、ミュートボタンを置く。出力デバイスの MASTER は `AUDIO OUT` の直下に専用チャンネルとして置く。通常の再生アプリと Vivaldi は、アイコン付きのチャンネルとして表示する。Vivaldi のマスターとタブは同じグループに置き、タブをインデントしてマスターの子として見せる。新しい通常アプリは2回連続の検出後に追加し、停止後も手動削除されるまで残すため、短い通知音のチャンネルが残りにくい。出力先の選択は、`AUDIO OUT` のプルダウンだけを表示する。

通常アプリのカードは、起動ごとに変わるオーディオセッションIDではなく実行プロセス名を識別子として再利用する。同じアプリを再起動した場合は、既存のカードへ状態を反映して重複表示を防ぐ。

出力デバイスのプルダウンは、ユーザーが選択したことのあるデバイスを最近使った順に先頭へ表示する。履歴はユーザーのローカルアプリデータに最大12件を保存する。

アプリケーションマニフェストで Per-Monitor V2 DPI 認識を宣言する。異なるスケール設定のモニター間へウィンドウを移動しても、Windows のビットマップ拡大によるぼやけを避ける。

有効な音量制御対象ではない NVIDIA の `nvcontainer` オーディオセッションは、ブラックリストで一覧から除外する。
